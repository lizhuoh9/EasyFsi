# HIBM-MPM Paper-vs-Code Phase 0

Reference: `Numerical Methods in Fluids - 2007 - Gilmanov - A hybrid immersed boundary and material point method for simulating 3D.pdf`.

Scope: compare the solver-level requirements in Gilmanov and Acharya Sections 2-4 against the current implementation in this checkout. Squid is only a validation case. The solver work belongs in `simulation_core`; `cases/squid_soft_robot/runner.py` may only select a coupling mode and report diagnostics.

Implementation constraint: every production HIBM-MPM solver operation must be Taichi-resident. IB search, inside/outside classification, normal reconstruction, pressure/velocity boundary rows, stress sampling, traction, and MPM force scatter must use Taichi fields/kernels. NumPy loops, `.to_numpy()`, `.from_numpy()`, or host round-trips may be used only for offline tests or one-time initialization, not for the runtime solver path.

## Phase 0 Conclusion

The current legacy projected/reduced coupling is not paper HIBM-MPM. It remains available only as `legacy_projected_reduced` so existing tests and diagnostics can still run, but it must not be labelled as sharp-interface HIBM-MPM.

The new solver mode name is `hibm_mpm_sharp`. The generic core sharp step now exists in `simulation_core`; `core_runner_available=true`, `case_runner_available=true`, and the 200-step explicit fine-nozzle probe is bounded. Generic sharp-step summary flattening lives in `simulation_core.hibm_mpm_sharp_step_summary`, and the squid case has a thin Taichi-field adapter that builds `HibmMpmSharpCouplingState` from MPM surface fields. The squid case runner consumes HIBM marker/full-stress summaries and fluid projection counters without requiring projected-IBM residual or force-probe reports. The sharp assembly applies internal IB nodes to a Taichi-resident HIBM obstacle mask before predictor/projection, no-slip residual sampling, and stress sampling. Case runners can therefore report IB counts, invalid projections, no-slip residuals, velocity-Dirichlet reconstruction validity/weights, pressure-Neumann row validity/gaps/transmissibility, region marker counts, marker forces, action-reaction residuals, and surface state updates without owning solver internals.

Current validation: the previous step-9 pressure-CG breakdown was caused by accepting invalid normal reconstructions as effective boundary rows. In the failing probe, many velocity Dirichlet rows had `projection_weight=0` while their pressure Neumann rows stayed active, so the paper's paired normal reconstruction assumption was broken. The solver now rejects velocity and pressure boundary rows unless the IB node lies on the boundary-to-interior-fluid normal segment, and it reports invalid reconstruction counts for both row types. The 200-step `rho=8x`, `grid-scale=4`, `fv_cg`, `projection_iterations=3000`, `fluid_substeps=1` probe in `_codex_sharp_phase5_probe_200step_segment_valid` completed with `completed_step_checks_passed=true`, `pressure_projection_cg_converged_all=true`, `pressure_projection_cg_breakdown_count=0`, `max_cfl=8.621022e-07`, `max_interior_divergence_l2=9.285781e-03`, tail marker participation, and final lip/outlet/downstream negative-z flow. This is still an explicit 200-step validation, not a full pressure-waveform run. A device-side CG direction restart trial was rejected because it let nonconverged pressure continue into marker traction and worsened the same step to `interior_divergence_l2=3.097604e-01` and `hibm_full_stress_max_abs_traction_pa=8.416800e+05`; it is not part of the implementation. A RHS-only natural-Neumann trial was also rejected because it produced a step-5 CFL blow-up; it is not part of the implementation. Edge-flux FV matvec, quadratic-form alpha denominator, and f64 CG work-vector trials were also rejected because targeted FV-CG tests or the same sharp probe turned red; they are not part of the implementation.

## Stability Gate

The completed-step stability gate must use the raw physical interface map, not a solver-relaxed or selected map. If `max_fsi_coupling_raw_interface_map_amplification > 1`, `fsi_physical_interface_map_stable` must be false for active multi-iteration coupling. Pressure projection failures are also hard failures: any row with `pressure_projection_cg_converged_all=false`, `total_pressure_projection_cg_converged_all=false`, or a positive pressure CG breakdown count must fail the numerical guard and completed-step checks.

## Legacy Boundary

`legacy_projected_reduced` means projected-IBM residual correction plus reduced main/tail region-pair reaction and IQN/Aitken fixed-point machinery. It is useful for diagnostics and regression comparison, but it is not paper HIBM-MPM. Both full-run and preflight summaries expose `fsi_coupling_mode_report` so this legacy mode is not relabelled as paper HIBM-MPM.

`hibm_mpm_sharp` means the future generic sharp-interface solver path:

```text
surface markers
-> IB node search
-> inside/outside classification
-> normal reconstruction
-> velocity Dirichlet no-slip
-> pressure Neumann matrix rows
-> fluid solve
-> full-stress traction
-> per-marker MPM external force
-> MPM surface update
-> next-step fluid boundary update
```
